# Структура папок в модах Dota 2 (VPK)

## 👤 Герои и Предметы (Где искать скины)
Герои распределены по нескольким папкам. Для полной очистки нужно удалить подпапку героя во всех этих местах:

### Основные модели:
- `models/heroes/<name>` — базовые модели.
- `models/items/<name>` — **ВАЖНО:** здесь лежат Арканы и многие косметические сеты (например, Аркана Шейкера).

### Материалы и частицы:
- `materials/models/heroes/<name>`
- `materials/models/items/<name>`
- `particles/units/heroes/hero_<name>`
- `kisilev_ind/models/<name>` (и другие кастомные папки типа `jxj`)

### Звуки:
- `sounds/weapons/hero/<name>`
- `sounds/vo/<name>`

## scripts/items/items_game.txt

`scripts/items/items_game.txt` — общий каталог предметов и косметики. Он может содержать ссылки на:
- модели: `models/...`
- материалы: `materials/...`
- частицы: `particles/...`
- кастомные папки модов: `kisilev_ind/...`, `8213/...`, `jxj/...`

Внутри VPK эти пути не являются Windows-путями. Это виртуальные игровые пути, которые Dota ищет в подключённых папках и VPK-архивах по приоритету.

Главный риск оптимизации: если оставить модовый `items_game.txt`, но удалить файлы, на которые он ссылается, обычные сеты неактивных героев могут стать невидимыми, потерять эффекты или показывать ERROR.

Для strict lightweight-сборки это не решается возвратом ассетов выключенного героя в VPK. Если герой не в `active_roster`, его ассеты должны быть удалены. Правильное решение — чистить или откатывать записи этого героя в `items_game.txt`, чтобы они не ссылались на удалённые модовые пути.

Но `items_game.txt` нельзя чистить как набор независимых блоков предметов. Внутри файла есть event/econ связи: `event_id`, `effects_item_def`, bundles, styles, rewards и другие секции могут ссылаться на item id, который на первый взгляд относится к выключенному герою или старому событию. Если удалить или заменить такой блок без проверки всех ссылок, Dota может упасть при инициализации schema. Реальный пример: `EVENT_ID_WINTER_MAJOR_2016` падал с `Unable to find effects_item_def 16844`.

Безопасный sanitizer должен сначала собрать ссылки по всему `items_game.txt`, затем сохранять все referenced id или корректно переносить связанные блоки вместе. Простая замена hero item-блоков на дефолтные блоки недостаточна.

## Проверенные провалы

Эти варианты уже проверялись и не должны использоваться как рабочий процесс:

1. **Вернуть ассеты выключенного героя в VPK.** Это может скрыть баг, но нарушает strict lightweight-правило: выключенного героя не должно быть в архиве.
2. **Заменить много item-блоков дефолтными блоками.** Dota падала при инициализации schema: `EVENT_ID_WINTER_MAJOR_2016` / `Unable to find effects_item_def 16844`.
3. **Заменить один блок Lion (`572`).** Запуск проходил, но визуальная проблема оставалась: сеты отсутствовали, иконки/эффекты продолжали приходить из модовых записей.
4. **Заменить все найденные блоки Lion и один блок Bounty Hunter.** Dota снова падала с `effects_item_def 16844`.

Вывод: для `items_game.txt` нельзя считать item-блок автономной единицей. Даже если блок выглядит связанным только с одним героем, его полная замена может нарушить старые event/econ связи или формат, на который рассчитывает Dota.

Правильное направление — точечная правка строк внутри существующего блока:

- оставить item id и структуру блока;
- оставить `event_id` и служебные econ-поля;
- не трогать bundle/style/reward/effects секции без доказанной необходимости;
- менять только пути на модели, иконки, эффекты и `asset_modifier`, которые ведут на удалённые модовые ассеты;
- каждый шаг проверять отдельным VPK и запуском игры.

## Нюансы имен
1. **Io (Wisp):** Использует `wisp`, `io`, а также `portal`, `cube`, `companion` (для Арканы-куба). Если удалить Portal-активы, Аркана будет ERROR.
2. **Phantom Lancer:** Может встречаться как `phantom_lancer` и `phantomlancer`.
3. **Sniper:** В некоторых местах встречается как `kardel`.
4. **Ложные срабатывания:** Короткое имя `io` может встречаться в именах `lion`, `furion`, `legion`. Нельзя фильтровать простым поиском подстроки; нужно сравнивать нормализованные имена папок и алиасы.
